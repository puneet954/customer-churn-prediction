# Customer Churn Prediction

An end-to-end machine learning project that predicts whether a telecom customer
is likely to **churn** (cancel their service), based on the
[Telco Customer Churn dataset](https://www.kaggle.com/datasets/blastchar/telco-customer-churn).

The project includes:
- A full training pipeline (data ingestion → preprocessing → model selection → saved model)
- A prediction pipeline for single or batch inference
- A **Streamlit web app** for interactive predictions

### App features

**Single Customer Prediction**
Fill in a form with a customer's details (contract type, tenure, charges,
services subscribed, etc.) and get an instant churn / no-churn prediction
with a churn probability score.

**Batch Prediction (CSV Upload)**
Upload a CSV containing multiple customers (a downloadable template is
provided in the app). The app validates the required columns, runs
predictions on every row, shows a summary of how many customers are
predicted to churn vs. stay, and lets you download the full results as a
CSV with two new columns: `Predicted_Churn` and `Churn_Probability`.

Required columns for batch upload:
```
gender, SeniorCitizen, Partner, Dependents, tenure, PhoneService,
MultipleLines, InternetService, OnlineSecurity, OnlineBackup,
DeviceProtection, TechSupport, StreamingTV, StreamingMovies, Contract,
PaperlessBilling, PaymentMethod, MonthlyCharges, TotalCharges
```
Extra columns (like `customerID` or `Churn`) are fine — they're ignored.

## Retraining the Model

If you change the data or the pipeline code and want to regenerate the
artifacts, run from the `project` folder:
```bash
python -m src.pipelines.train_pipeline
```

This will:
1. Read the raw CSV from `notebook/data/`
2. Split it into train/test sets and save them to `artifacts/`
3. Fit a preprocessing pipeline (median/mode imputation, scaling, one-hot
   encoding) and save it as `artifacts/proprocessor.pkl`
4. Train and compare several classifiers — Random Forest, Decision Tree,
   Gradient Boosting, Logistic Regression, XGBoost, CatBoost, AdaBoost
5. Save the best-performing model (by test accuracy) to `artifacts/model.pkl`

Logs for each run are saved to `logs/<timestamp>.log`.

## Model

- **Target:** `Churn` (encoded as `1` = Yes, `0` = No)
- **Numerical features:** `MonthlyCharges`, `TotalCharges`, `tenure`, `SeniorCitizen`
- **Categorical features:** `gender`, `Partner`, `Dependents`, `PhoneService`,
  `MultipleLines`, `InternetService`, `OnlineSecurity`, `OnlineBackup`,
  `DeviceProtection`, `TechSupport`, `StreamingTV`, `StreamingMovies`,
  `Contract`, `PaperlessBilling`, `PaymentMethod`
- **Best model (current artifacts):** Logistic Regression, ~82% test accuracy
